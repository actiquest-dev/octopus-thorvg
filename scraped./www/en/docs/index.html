ANIMA — Technical Documentation

Adaptive Neural Interface for Motion & Animation

Real-time generative avatar animation and interactive vector worlds with neural control

Generative AI Exits Render Mode

All generative AI today works the same way: prompt → wait → file. Midjourney produces an image in a minute. Sora produces video in minutes. Impressive, but it's rendering. You get a finished artifact.

ANIMA is the first system where generation happens in real-time. Not "create me a scene" — a scene that exists and changes while you're watching it. 30 frames per second of continuous generation.

The difference between a photograph and a window.

Why It Hasn't Been Done

Economics didn't work. Real-time video generation costs $0.30 per minute per user. That's $18 an hour. Fine for enterprise demos, not for consumer products.

We rethought what exactly the neural network should generate. Not pixels — what's behind them. This cut costs 100x and unlocked markets that were previously uneconomical.

What Changes

Interactive media finally becomes real. Netflix tried with Bandersnatch — pre-rendered branches, choose your ending. That's a tree, not a world. We generate the response to anything, in real-time. Every viewer sees their own version.

AI companions — a $2B market growing 50% annually — are stuck in text chat. Character.ai, Replika, dozens of clones. 20 million users messaging bots because the alternative costs too much. We give those bots a face, a voice, and a world around them.

Adult content — $100B annually — adopts every new media technology first. VHS, streaming, VR. Real-time generative will be next.

Why Now

Three things converged. Models learned to work fast — single pass, not iterative steps. Browsers learned to render complex graphics. Voice models became indistinguishable from human.

Separately, evolution. Together, a new product category.

What's Next

First product — interactive story platform. Anime where the viewer participates. Then creator tools: anyone builds an interactive character without animation skills.

But that's just the start. When generation becomes real-time, everything changes. Every banner personalized. Every game NPC unique. Every training course adaptive.

We're not improving rendering. We're eliminating it.

Audio Engine

Multi-layer audio architecture supporting real-time speech synthesis, emotion processing, and spatial audio.

Why Own Stack (Not Hume)

  • Cost/min: $0.002–0.005 vs $0.05–0.10 (Hume)
  • Control: Full ownership vs Limited
  • Latency: Local vs +50ms API call
  • Voice clone: Supported vs Not available

Audio Layers

  1. Voice (TTS/Voice Clone) - XTTS v2, StyleTTS2, Fish Speech
  2. Emotion - Text classifier (10ms), Prosody extraction
  3. SFX - Script triggers, Procedural sounds, Library foley
  4. Ambience - Scene-based, Dynamic weather/time
  5. Music - Adaptive score, Stems mixing

ANIMA SDK — Custom Animation Engine

Open-source WebGL2 engine optimized for hybrid neural/parametric rendering.

Why Not Live2D

  • Proprietary SDK with license costs
  • Limited extensibility for neural blending
  • Unpredictable performance and closed format coupling
  • ANIMA SDK is MIT licensed and fully open

SDK Modules

  • Asset Pipeline: Import Live2D (.moc3), Spine, PSD/SVG → Convert to .anima
  • Deformation Engine: Mesh skinning (GPU), Blend shapes, Bone IK, Neural texture injection
  • Render Pipeline: Layer compositor, Neural face overlay, Particle system, Post-processing
  • Runtime: Parameter interpolation, Animation state machine, Event system, Performance profiler

Performance Contract

  • Target: 30 FPS perception, <50ms end-to-end latency
  • Adaptive budget: Reduce particles, LOD paths, disable effects — keep motion smooth

Streaming Protocol — JSON Canvas Instructions

ANIMA streams rendering instructions, not video frames.

Message Format (Delta Encoding)

{
  "frame_id": 1235,
  "delta": true,
  "face.mouth_open": 0.65,
  "face.emotion_blend.happy": 0.7,
  "body.position": [0.5, 0.6],
  "audio_sync.viseme_id": 3
}

Typical frame size: 200–500 bytes (delta) vs 2KB (full state)

Transport Options

Protocol Latency Use Case
WebSocket 10–30ms Default, reliable
WebRTC DataChannel 5–15ms Ultra-low latency
SSE 30–50ms Fallback, simple

Why This Works

  • Server: Lightweight inference (~30ms), no video encoding
  • Bandwidth: ~30 KB/s (JSON) vs 2–5 Mbps (video)
  • Client: Full control over rendering, local GPU acceleration
  • Scalability: 400 users per A100 (vs 5–10 for video)

Vector Worlds — Generative Scenes with Physics

ANIMA generates entire interactive vector worlds in real-time. Server handles physics and scene generation; client receives lightweight vector commands.

Architecture

Backend (full compute) runs:

  • Scene Generator: Neural or procedural scene generation
  • Physics Engine: Deterministic 60Hz simulation
  • Vector Compositor: Path/shape/transform streaming (~20KB/frame)

Client renders only: Canvas2D or WebGL path API applied to streamed transforms.

Scene Generation Modes

Mode A: Procedural (Fast, Controllable)

  • Templates: underwater_cave, neon_city, abstract_space
  • Parameters: depth, density, light_rays, building_height, etc.
  • Fast rendering with precise control

Mode B: Neural (Creative, Surprising)

  • Text-to-vector generation (VectorFusion-style)
  • Latency: 200–500ms for initial generation
  • Then procedural animation with low latency

Physics on Backend

  • Rigid Bodies: Collision detection, gravity, joints
  • Soft Bodies: Cloth, rope, jelly objects
  • Particles: 10,000 max per frame (sparks, dust, rain, leaves)
  • Simplified Fluids: Metaballs or height-field water

Format & Efficiency

Delta Encoding: Only changed objects are sent.

Compression: MessagePack + LZ4 → ~10–30 KB/frame typical.

Client Rendering: Canvas2D Path API (simple) or WebGL (high complexity).

Performance Budget

  • Physics step: 5ms (60Hz internal)
  • Scene update: 2ms
  • Vector serialize: 3ms
  • Network: 10ms (buffered)
  • Total backend: ~20ms
  • End-to-end: ~50ms (30 FPS comfortable)

Complexity Limits

  • Rigid bodies: 500 max
  • Particles: 10,000 max
  • Vector paths: 2,000 max
  • Gradient fills: 200 max

Parameter Extraction Pipeline

From video/stream to 170D parameters:

  • Audio processing: Phoneme alignment (MFA) → timing, Emotion classifier → emotion (8D)
  • Video processing: Face landmark detection (MediaPipe) → mouth, eyes, brows (~40D), Pose estimation → body params (~30D)
  • Result: (audio_features, emotion, visemes) → (170D params) per frame

Synthetic Data Generation

For unlimited training data:

  1. Sample emotion trajectory (random walk)
  2. Generate text matching emotion
  3. TTS → audio waveform
  4. Rule-based animator → baseline params
  5. Add noise/variation
  6. Model learns to exceed baseline quality

Advantage: infinite data, controlled conditions. Disadvantage: ceiling on quality (rules-based ground truth)

Model Architecture Summary

Complete System Pipeline

OFFLINE (Content Generation): Script → TTS → Audio → Phoneme Align → Emotion Extract → (audio_feat, emotion, viseme)

REAL-TIME (Inference): (audio_feat, emotion, viseme, char_id, prev) → Context Encoder (15M) → Z (256D) → Flow Decoder (2M) → Params (170D) → JSON → WebSocket

CLIENT: JSON → Interpolation → SDK → WebGL Canvas

Inference Timeline

  • T=0ms: Audio features ready
  • T=8ms: Context Encoder complete → Z (256D)
  • T=20ms: Flow Decoder complete → Params (170D)
  • T=22ms: JSON encoded, sent via WebSocket
  • T=32ms: Client receives JSON (network ~10ms)
  • T=33ms: Interpolation + SDK apply
  • T=38ms: WebGL render complete → Frame displayed

Total: ~38ms end-to-end (50+ FPS capable)
