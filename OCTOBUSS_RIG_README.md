# Octobuss SVG Rig — как устроен и как править

Осьминог-космонавт с сгибающимися щупальцами, липсинком, эмоциями, бровями и
спецэффектами. Персонаж перерисован попиксельно с референса `octobuss.source.png`.

```
generate_octobuss_rig.py  ──запуск──▶  octobuss.rigged.svg  ──cp──▶  frontend/octobuss.rigged.svg
        (арт + метаданные)                                                    │ fetch
                                                                              ▼
frontend/octobuss_avatar.js  ◀──── рантайм-риг: гнёт щупальца, рот, глаза, FX
        │ реализует интерфейс avatar-proxy
        ▼
frontend/index.html (кнопка «🦑 SVG Rig») ◀── команды от Gemini/timeline_sync.py
```

## Файлы

| Файл | Что это |
|---|---|
| [generate_octobuss_rig.py](generate_octobuss_rig.py) | **Источник арта.** Генерирует SVG: палитра, спайны щупалец, лицо, шлем, воротник, тело |
| [octobuss.rigged.svg](octobuss.rigged.svg) | Сгенерированный персонаж (не править руками — перезапишется!) |
| [frontend/octobuss_avatar.js](frontend/octobuss_avatar.js) | **Рантайм.** Класс `OctobussAvatar`: FK-кости, позы, липсинк, эмоции, брови, частицы |
| [frontend/test_octobuss.html](frontend/test_octobuss.html) | Стенд: кнопки всех поз/эмоций/действий/визем |
| [octobuss.rigged.preview.html](octobuss.rigged.preview.html) | Автономное превью (CSS-анимация, без рига) |
| [docs/octobuss-rig-api.md](docs/octobuss-rig-api.md) | Протокол команд для ИИ/бэкенда |

**Цикл правки:**

```bash
# 1. правим generate_octobuss_rig.py
python3 generate_octobuss_rig.py          # 2. пересборка SVG
cp octobuss.rigged.svg frontend/          # 3. в фронтенд
# 4. F5 на http://localhost:8081/test_octobuss.html
```

## Как это работает

### Щупальца (7 шт.)

Каждое щупальце в SVG — группа `<g id="tentacle_..." class="tentacle">` с метаданными:
`data-spine` (контрольные точки осевой линии), `data-w0`/`data-w1` (толщина у корня и
кончика), `data-taper` (кривая сужения), `data-side` (сторона присосок/тени).

Рантайм разбирает спайн на «кости» (длины + углы сегментов) и **каждый кадр**
пересобирает всю кожу: сглаживает спайн сплайном Катмулла-Рома, строит контур
смещением по нормалям, тень по нижней кромке, расставляет присоски (`class="t-sucker"
data-t=...`), пятна (`t-spot`) и манжеты (`t-band`) вдоль текущего изгиба.
Поэтому щупальце гнётся по всей длине, а декор следует за ним.

Параметры позы одного щупальца: `rot` (поворот, распределяется по 2–4-му суставам —
корень приращён к телу), `curl` (до-закрутка завитка), `straighten` (разгибание 0..1),
`stretch`, `swingAmp/swingSpeed/swingPhase` (циклический мах), `waveMul` (амплитуда
идл-волны).

**Ограничитель видимости**: перед применением позы риг просчитывает, куда попадёт
кончик; если за стекло шлема, за тело или за край холста — изгиб автоматически
ужимается до видимой зоны (бинарным поиском). Любая поза от ИИ безопасна.

### Рот и липсинк

Путь рта (`#mouth_open`) и язык (`#tongue`, `#tongue_hi`) — параметрический морф,
пересобираются каждый кадр из `open/width/round/curve`. Виземы `REST A E I O U`
задают форму, трек `jaw` (0..1) — степень открытия. Планировщик пакетов
`audio_sync` совместим с `timeline_sync.py` (выравнивание по `getAudioPlayheadS`,
догон при опоздании).

### Лицо

Зрачки (`#pupil_l/r`) двигаются под `clipPath` — не выходят из орбиты. Большой блик
(`#hi_l/r`) лежит **поверх века** — виден даже при моргании. Веки (`#lid_l/r`) —
эллипсы цвета кожи, сдвигаются сверху (моргание, прищур, эмоции). Брови
(`#brow_l/r`) видимы всегда, состояние своё у каждой эмоции, позы и эффекта.

### Слои (сзади → вперёд)

`up_l, far_r, up_r` (за телом) → `body` (мантия с перепонкой — заполняет щели
при изгибах) → `band_l, front_l, front_r, front_c` → тень воротника →
заднее кольцо воротника → голова+лицо → стекло → передняя труба воротника → `fx_layer`.

## Как менять

### Цвета

Вся палитра — константы в начале `generate_octobuss_rig.py`:

```python
MAIN   = "#B678E4"   # основной фиолетовый
SHADE  = "#8848B8"   # тень (низ форм)
DARK   = "#683888"   # борозды/брови
HI     = "#D8B8F8"   # светлые пятна
SUCK   = "#F888A8"   # присоски и щёки
GRAY   = "#585858"   # воротник/манжеты
MOUTH  = "#682838"   # рот
TONGUE = "#E86888"   # язык
...
```

Поменяли → пересобрали → готово. Рантайм цвета не трогает (только геометрию),
поэтому менять цвета можно и прямо в `frontend/octobuss.rigged.svg` — но при
следующей пересборке правки затрутся, лучше в генераторе.

Исключение: цвета **частиц FX** зашиты в `octobuss_avatar.js` → `FxSystem._make()`
(сердце `#F888A8`, искра `#FFE066`, капля `#8AD4FF`, кольцо `#FFD166` и т.д.).

### Форма и положение щупалец

Словарь `TENTACLES` в генераторе. Формат:

```python
"tentacle_up_r": ("имя",
    [(1470,1235),(1510,1170), ...],   # спайн: точки осевой линии, корень → кончик
    145, 52,        # толщина у корня / у кончика
    1,              # сторона присосок (+1/-1)
    [0.04,0.15,...],# присоски: позиции t вдоль спайна (0=корень, 1=кончик)
    [0.20,0.45],    # светлые пятна: t
    0.33,           # манжета: t (None = нет)
    1.2),           # taper — кривая сужения
```

Сдвинули точку спайна → щупальце изменило форму, присоски/тень/манжета
перестроятся сами. Кончик закручивается последними 3–5 точками спайна
(радиус завитка держите больше половины толщины в этом месте, иначе контур
сложится). Координаты — в системе viewBox 2048×2048.

⚠️ Если меняете спайн сильно, проверьте `CURL_DIR` в `octobuss_avatar.js` —
направление «естественного» завитка (+1 = по часовой на экране).

### Лицо: глаза, щёки, рот, брови

Всё в `head_group()` генератора — обычные координаты SVG:

- **глаза**: центры склер (799,747)/(1195,765), зрачков (821,749)/(1179,767);
- **щёки**: `cheek_l` (730,886), `cheek_r` (1259,912);
- **рот**: базовый путь `#mouth_open` (только вид ДО инициализации рига);
- **брови**: пути `#brow_l/r` (толщина = `stroke-width`).

⚠️ **Связанные константы в `octobuss_avatar.js`** — при сдвиге лица поменяйте и их:

| Что сдвинули | Где поправить в JS |
|---|---|
| зрачки | `this.pupils = [{cx: 821, cy: 749}, {cx: 1179, cy: 767}]` |
| рот | `MouthRig`: `this.cx = 986; this.topY = 862` |
| глаза (clipPath) | `eyeClipL/R` в defs генератора |
| стекло шлема | `tipVisible()`: эллипс `(1041,722) / 525×487` |
| тело-мантия | `tipVisible()`: эллипс `(1024,1330) / 400×245` |

### Шлем, воротник, тело

- **стекло**: `helmet_group()` — эллипс (1041,722) rx 508 ry 470, дуги бликов
  задаются углами (`range(196, 254, 4)` — верхне-левый серп);
- **воротник**: `collar_back_group()` (светлая верхняя грань, за головой) и
  `collar_front_group()` (передняя труба с провисающим верхним краем, поверх стекла);
- **тело**: `body_group()` — путь мантии: верх прячется за воротником, нижняя
  кромка — провисающие дуги (`Q`) между корнями щупалец. Если щупальца
  разъезжаются и сквозит фон — расширьте этот путь.

### Позы

Словарь `POSES` в `octobuss_avatar.js`. Поза = параметры на щупальце + опционально
`__brows` (состояние бровей) и `__fx` (`squash` — сжатие тела, `squint` — прищур):

```js
my_pose: {
    __brows: [{ op: 1, rot: -6, dy: -14 }, { op: 1, rot: 6, dy: -14 }],
    __fx: { squash: 0.5, squint: 0.2 },
    tentacle_up_r: { rot: -0.4, straighten: 0.7, curl: -0.5,
                     swingAmp: 0.2, swingSpeed: 5 },
    // не указанные щупальца вернутся в rest
},
```

Проверка: `test_octobuss.html?pose=my_pose` (появится в списке `avatar.listPoses()`).
Маппинг «действие → поза» — объект `map` в `activateAction()`.

### Эмоции

Таблица `EMOTIONS`: `lid` (прикрытие век 0..1), `pupil` (масштаб зрачков),
`cheek` (яркость щёк), `curve` (изгиб уголков рта), `openBias` (приоткрытие),
`pose` (фоновая осанка), `brows`. Пример: `sad: { pose: "droop_sad", lid: 0.42, ... }`.

### Идл-анимация и мелочи

- бегущая волна щупалец: коэффициент `0.10` и частота `2.2` в `Tentacle.buildSpine()`;
- дыхание головы: амплитуды в блоке `— голова:` в `_tick()`;
- частота моргания: `dt / 4.2` (раз в ~4 с) в `_tick()`;
- диапазон взгляда: `dx = st.gaze.x * 30; dy = st.gaze.y * 24`;
- скорость перехода поз: `durationMs` в `setPose` (τ = duration/4);
- частицы: `FxSystem` (формы в `_make`, физика в `update`), вызовы —
  `triggerFxFromAction()`; моушены (спин/баунс/буп) — объект `this.motion`.

## Тестирование

```bash
python3 -m http.server 8081 --directory frontend   # если не запущен app.py
```

- Стенд: `http://localhost:8081/test_octobuss.html` — все кнопки; зрачки следят
  за курсором; «▶ проиграть фразу» — синтетический пакет `audio_sync`.
- Query-параметры для скриншотов/автотестов:
  `?pose=X`, `?emotion=Y`, `?viseme=O&jaw=0.9`, `?action=sfx_heart`,
  `?gaze=1,0`, `?demo=1` (липсинк).
- В основном демо: `index.html` → кнопка **«🦑 SVG Rig»**. Аватар получает те же
  команды, что и остальные (через `window.octopusAnimator`).

## Правка в Illustrator — коротко

Можно менять: цвета, шлем, воротник, щёки, блики (статичные части).
Бессмысленно: контуры щупалец и рта — рантайм пересобирает их каждый кадр.
Опасно: Illustrator при экспорте вычищает `data-*` атрибуты и переименовывает
`id` — риг сломается. Правьте в AI только как референс, переносите изменения
в `generate_octobuss_rig.py` руками. И помните: SVG — генерируемый артефакт.
