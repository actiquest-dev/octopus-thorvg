# Зеркалирование сайта

Этот скрипт скачивает указанную страницу и все связанные ресурсы (CSS, JS, изображения, дополнительные HTML-ссылки) внутри одного домена, переписывает ссылки под локальную структуру и сохраняет результат в папке `output/<хост>/<путь>`.

## Что делает

- Поддерживает `href`, `src`, `srcset`, `poster` и другие атрибуты для картинок, скриптов, стилей и iframe-ов.
- Ограничивается файлам внутри того же домена и по умолчанию по пути `/en` (то есть не уходит в чужие секции).
- Рекурсивно обходит внутренние ссылки в пределах `--max-depth` (по умолчанию 3), так что вы можете получить статичный снимок раздела.
- Сохраняет файлы по структуре вида `output/experience.slap-apps.de/en/index.html`.

## Установка зависимостей

```bash
pip install requests beautifulsoup4
```

## Пример запуска

```bash
python scripts/mirror_site.py https://experience.slap-apps.de/en --output ./scraped
```

После этого `./scraped/experience.slap-apps.de/en/index.html` и все ассеты будут лежать локально. Чтобы посмотреть страницу, можно поднять простой сервер рядом с папкой:

```bash
cd scraped
python -m http.server 8000
# а потом открыть http://localhost:8000/experience.slap-apps.de/en/
```

## Параметры

- `--max-depth N` — насколько глубоко следовать ссылкам (`href`). Чем больше глубина, тем больше файлов, но и дольше загрузка.
- `--output DIR` — куда сохранять собранный сайт. Если папка не существует — создаётся.

## Ограничения

- Парсится только HTML и ресурсы внутри указанного домена. Внешние CDN-файлы не скачиваются и остаются как есть (все их `href/src` остаются оригинальными).  
- Скрипт не выполняет CSS/JS, поэтому необычные динамические вставки (например, URL, генерируемые через JS) не будут обнаружены.  
- Встроенные ссылки с фрагментами (`#`) не дублируются — фрагмент отбрасывается.

Если нужно гибче (например, скачивать только изображения или использовать авторизацию), можно подправить `scripts/mirror_site.py` или попросить дополнение.
