ANIMA — Adaptive Neural Interface for Motion & Animation

Real-time JSON control stream for avatars and vector worlds. The server orchestrates physics, inference and emotion, the client renders a living experience with deterministic timing.

Executive summary

ANIMA streams JSON instructions (~0.2–2 KB/frame) instead of video, enabling 30 FPS perception with 50–100 ms end-to-end latency. Control targets both stylized avatars and physics-first vector worlds; inference happens server-side, rendering happens client-side.

  • Dual product tracks: Avatar Mode (stylized face/body/effects) and Vector Worlds Mode (interactive physics scenes).
  • Neural parameter synthesis + procedural layer ensure smooth micro-motions (blink, breathing, sway).
  • Economic edge: ~$0.002/min streaming plus 100–200 concurrent users per GPU vs. video’s $0.20+ and 5–10 users.

System architecture

┌───────────────────┐ WebSocket / WebRTC ┌───────────────────┐
│ ANIMA BACKEND │ ───────────────────── │ CLIENT RUNTIME │
│ (voice/emotion, │ │ (JSON decode, │
│ inference, JSON │ │ ANIMA SDK render)│
└───────────────────┘ └───────────────────┘

The backend consumes Hume EVI audio/emotion, context, and interaction. It runs flow-based neural nets to generate parameter deltas + world ops, compresses them, and streams via WebSocket. The client merges JSON deltas, drives animation/physics via the ANIMA SDK, and renders WebGL2/Canvas2D without any video encoding.

Server

Receives inputs, runs neural inference, emits JSON control + world ops, optional authoritative physics.

Client

Parses deltas, validates, interpolates, applies to scenegraph + physics, renders frames locally.

Inputs & ANIMA SDK

Inputs span voice/prosody (Hume), text/context (scene intent, avatar ID, presets), tactile interactions (click/touch, physics triggers, Ink tags), and gaze + attention (ANIMA-SENSE).

The ANIMA SDK organizes:

  • Asset pipeline: Live2D/Spine/PSD/SVG → .anima, build meshes/rigs/materials.
  • Animation runtime: layered state store, JSON merge/validation, interpolation + easing, profiler.
  • Deformation: GPU skinning, blend shapes, IK, constraints.
  • Optional physics: rigid bodies, soft bodies, particles tied to deterministic simulation.
  • Rendering: WebGL2 compositor, vector draw ops, postFX, render-to-texture.

Models, controls & timeline

Model stack
  • ANIMA-AVATAR: 170‑dimension parameter generator (mouth, eyes, brows, body, hands, FX).
  • ANIMA-WORLD: world control stream (apply_force, spawn, constraints, camera moves).
  • ANIMA-PHYS: authoritative physics producing deterministic transforms.
  • ANIMA-DRAW: render compiler that handles LOD, budgets, pseudo-volume scenegraph.
  • ANIMA-SYNC: timeline manager buffering multi-stream inputs.
  • ANIMA-SENSE: gaze/IMU perception overlay that feeds back intents without mutating physics.
Clock invariants
  • World Tick (60Hz) → physics + deterministic storage.
  • Render Tick (30 FPS perception) → interpolation + drawing, never blocks physics.
  • Sense Tick (10‑120Hz) → gaze, blink, IMU, audio, yields perception overlays.

State layers include Hard (physics), Visual (vector scenegraph + transforms), and Perception (local overlays). Streams follow snapshot + delta contracts for replayability.

Audio layers & platform use cases

Audio stack (v3.0)

  • Layer 2 Emotion: text + prosody classifiers (BERT, Wav2Vec) drive viseme timing + micro-sounds.
  • Layer 3 SFX: script triggers + procedural cloth/footstep foley.
  • Layer 4 Ambience: scene-based atmospheres.
  • Layer 5 Music: adaptive score with stem mixing keyed to tension.
